# CONTEXT.md Compiler — Deterministic Algorithm

## Purpose

Generate `.aibridge/CONTEXT.md` — a single markdown file that every agent reads first. It must be deterministic, bounded, and human-readable.

## Output Template

```markdown
# Project Context — {projectName}

> Auto-generated by AiBridge. Do not edit manually.
> Last updated: {timestamp}

## Project

- **Name**: {projectName}
- **Repo**: {repoPath}
- **Schema version**: {schemaVersion}
- **Last sync**: {lastSyncAt}

## Active Agents

| Agent | Kind | Config |
|-------|------|--------|
{for each agent in bridge.json.agents}
| {name} | {kind} | {configPath} |
{end}

## Task Summary

| Status | Count |
|--------|-------|
| Pending | {pendingCount} |
| In Progress | {inProgressCount} |
| Done | {doneCount} |

### In-Progress Tasks

{for each task where status == "in_progress", sorted by updatedAt desc, limit 10}
- **{title}** — assigned to {agentId || "unassigned"} (priority: {priority})
{end}

### Pending Tasks

{for each task where status == "pending", sorted by priority desc then createdAt asc, limit 10}
- **{title}** — assigned to {agentId || "unassigned"} (priority: {priority})
{end}

{if remaining > 0}
+{remaining} more pending tasks
{end}

## Recent Activity

{for each log entry, sorted by timestamp desc, limit 10}
- [{timestamp}] **{agentId}** {action}: {description}
{end}

{if remaining > 0}
+{remaining} more entries
{end}

## Open Handoffs

{for each handoff not yet acknowledged, sorted by timestamp desc, limit 5}
- **{fromAgentId} → {toAgentId}**: {description} ({timestamp})
{end}

## Unread Messages

{for each message where acknowledged == false, sorted by severity desc then timestamp desc, limit 5}
- [{severity}] **{fromAgentId}** → {toAgentId || "all"}: {content}
{end}

## Recent Decisions

{for each decision where status == "accepted", sorted by timestamp desc, limit 5}
- **{title}**: {summary}
{end}

## Active Conventions

{for each convention, sorted by addedAt asc}
- {rule}
{end}

## Suggested Next Actions

{for each active agent}
### {agentName}
{generated suggestions}
{end}
```

## Section Limits (Truncation)

| Section | Max items | Sort order |
|---------|-----------|------------|
| In-progress tasks | 10 | updatedAt desc |
| Pending tasks | 10 | priority desc, createdAt asc |
| Recent activity | 10 | timestamp desc |
| Open handoffs | 5 | timestamp desc |
| Unread messages | 5 | severity desc, timestamp desc |
| Recent decisions | 5 | timestamp desc |
| Conventions | unlimited | addedAt asc |

When truncated, append `+{N} more` at the end of the section.

## Suggested Next Actions — Generation Logic

For each active agent, generate up to 3 suggestions based on:

1. **Assigned tasks**: If the agent has `pending` tasks, suggest "Start working on: {task.title}".
2. **Open handoffs TO this agent**: If a handoff targets this agent, suggest "Review handoff from {fromAgent}: {description}".
3. **Unread messages TO this agent**: If messages are unaddressed, suggest "Read message from {fromAgent}".
4. **Idle detection**: If the agent has no pending tasks, no handoffs, and no messages, suggest "No pending work — available for new assignments".

## Token Budget Guidelines

Target total output: **~2,000 tokens** (approximately 1,500 words). This fits comfortably within the system prompt of most LLM agents while leaving room for the agent's own instructions.

If the output exceeds the budget:
1. Reduce log entries from 10 → 5
2. Reduce pending tasks from 10 → 5
3. Truncate decision summaries to 80 characters
4. Remove "Suggested Next Actions" section entirely as last resort

## Determinism Rules

- All timestamps in UTC ISO 8601 format.
- Sort order is stable (secondary sort by `id` when primary sort values are equal).
- Section order is fixed — never reorder sections.
- Empty sections render as `_(none)_` rather than being omitted.
- Markdown heading levels are fixed (## for sections, ### for sub-sections).

## Implementation Notes

The compiler reads from:
- `.aibridge/bridge.json` — project metadata and agents
- `.aibridge/tasks/*.json` — all task files
- `.aibridge/logs/*.jsonl` — all log files
- `.aibridge/handoffs/*.json` — all handoff files
- `.aibridge/messages/*.json` — all message files
- `.aibridge/decisions/*.json` — all decision files
- `.aibridge/CONVENTIONS.md` — parsed as a list of rules

Output is written to `.aibridge/CONTEXT.md` atomically (write to temp file, then rename).
